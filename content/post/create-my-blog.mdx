## 博客的技术选型

我的毕业设计题目是一个论坛，开题时打算完成后面再魔改成博客作为我自己的个人站点，当时选取的是 Nextjs 的 SSR+Koa 后端的方案，文章和评论系统采用的 quill 富文本编辑器，
后来发现某位大佬的博客使用的 SSG 和 markdown 方案才顿时感觉路走偏了，奈何大佬的博客并不开源但其文章也有大致描述自己的实现，那么干脆参考这个方案写一个好了。

### 2.部署方案

大二时薅羊毛一百不到买了十年的轻量应用服务器，放着不用就太浪费了，用来挂个博客绰绰有余，因此也不需要挂到 vercel 或者 githubPages 了。

之前两家实习过的公司都有自动化 git Action 来进行 CI/CD ，虽然之前没有接触过但为了摆脱手动部署的繁琐还是先把自动化流程打通,参考

### 3.布局及样式

#### 响应式布局方案

对于博客界面，响应式是肯定要做的，界面整体打算做三栏响应式布局，pc 上展示为三栏，平板上展示为两栏，移动端展示单栏。侧边栏定宽，内容栏自适应，使用媒体查询和 flex 布局实现界面适配。

#### CSS 方案

毕业前的最后一段实习接触到了 tailwindCSS 这种原子化 CSS，不需要编写样式名，不用遵守又臭又长的 BEM 规范，不用关心一层套一层的样式优先级覆盖，简直爽到嗨。这里不必多说
直接选用 tailwindCSS，强烈推荐尝试原子化 CSS，绝对是爽到上瘾的程度。
